# 📄 Hướng Dẫn Quản Lý Tài Liệu Báo Cáo
**Thư mục: Báo cáo Bài tập lớn (BTL)**

Thư mục này được sử dụng độc quyền để lưu trữ các phiên bản báo cáo của dự án gửi cho Giảng viên. Nhằm đảm bảo tính thống nhất và chuyên nghiệp khi làm việc nhóm, các thành viên vui lòng tuân thủ các quy tắc dưới đây.

## 🗂 Các Loại File Báo Cáo

### 1. File Word (`.docx`) - Bản Thảo (Draft/Editable)
- **Mục đích:** Là file gốc để các thành viên trong nhóm có thể mở lên chỉnh sửa, cập nhật nội dung, dán hình ảnh mô hình Star Schema, ảnh chụp màn hình CUBE, và viết nội dung phân tích.
- **Cách sử dụng:** Khi được phân công viết báo cáo, bạn hãy tải file này về để làm. **Lưu ý quan trọng:** Luôn bật chế độ *Track Changes* hoặc ghi rõ trong nhóm khi có ai đó đang sửa file để tránh tình trạng conflict (ghi đè) nội dung của nhau.

### 2. File PDF (`.pdf`) - Bản Nộp Cuối (Final Submission)
- **Mục đích:** Là phiên bản chốt cuối cùng, không thể chỉnh sửa, dùng để **nộp trực tiếp cho Giảng viên** trên hệ thống học tập.
- **Cách sử dụng:** Sau khi chốt xong nội dung trên file Word, Trưởng nhóm (hoặc người phụ trách báo cáo) có trách nhiệm `Save As` sang định dạng PDF và đẩy lên thư mục này. File PDF giúp giữ nguyên định dạng (font chữ, hình ảnh, lề) trên bất kỳ máy tính nào của Giảng viên khi mở ra chấm điểm.

---

## 💡 Quy Tắc Đặt Tên File (Naming Convention)
Để tránh nhầm lẫn giữa các phiên bản báo cáo (VD: `Bao_cao_final_1`, `Bao_cao_final_cuoi_cung`), nhóm thống nhất cách đặt tên file như sau:

**Cú pháp chuẩn:** `[Tên Môn Học] - [Mã Nhóm] - Báo Cáo BTL - [Phiên bản/Ngày]`
- **Ví dụ File đang soạn:** `KDL - Nhom07 - Bao Cao BTL - Draft_20260529.docx`
- **Ví dụ File nộp cuối:** `KDL - Nhom07 - Bao Cao BTL - Final.pdf`

*(Hãy đảm bảo file PDF cuối cùng của nhóm chứa đầy đủ bìa, mục lục, nội dung, danh sách thành viên phân công công việc, và kết luận trước khi nộp nhé! Chúc nhóm đạt điểm cao!)*
